什么是Spring boot 的默认资源映射?

当访问一个网站的时候的相关资源，比如css文件，静态html文件，js文件等。应该去哪里找？ 比如http://localhost:8080/css/main.css 当你访问这个资源的时候，服务要去后台服务器的哪个 文件夹里面找呢？默认资源映射，就是指的是这个css/man.css 文件要去服务器的哪个地方去找。 spring boot 的默认资源映射，是将/** 映射到 /static （或/public、/resources、/META-INF/resources） 那么，/css/main.css 就是 /**的一种，也就是说会到 static/文件夹下找这个 css/main.css 文件。

spring mvc Suffix match（后缀匹配） 问题

后缀匹配(Suffix match)在spring mvc 中是默认开启的，比如你把你的控制器映射到/person,也就是@Controller("/person")，也就默认 映射到/person.*，这里扩展的名字(.* 这个部分)可以用来解释出请求数据的类型，比如.json 就表示，请求的是一个json类型的数据，于是。 在服务器响应的时候，返回类型就是application/json。但是在spring boot 2.0.0中这个给后缀匹配被关闭了。如果你想明确给客户端返回特定的数据 类型，可以在映射控制器注解中这么写 @GetMapping(path = "/pets/{petId}", produces = "application/json;charset=UTF-8")，这就表示我服务器 响应的数据类型是"application/json"的.
